# Security

Please do not report private credentials through public issues.

If you find a leaked secret, unsafe default, or instruction that could expose private data, open a private vulnerability report if GitHub offers that option for this repository. If it is not available, contact the maintainer through a non-public channel.

When contributing, do not include:

- API keys, access tokens, cookies, session files, passwords, or private keys
- personal account names or emails
- absolute local paths
- browser profiles or exported login state
- platform balances, billing notes, or private provider configuration

If a skill needs a secret at runtime, describe the environment variable or secret-store lookup name. Do not put the value in the repository.
