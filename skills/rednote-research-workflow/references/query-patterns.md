# Query Patterns

Use this reference when building searches.

## Mix query types

Use several small query groups instead of one broad query.

### Topic query

Names the domain:

- research poster
- data competition
- design portfolio
- course project

### Audience query

Names the community:

- undergraduate
- graduate applicant
- designer
- product manager
- contest participant

### Output query

Names the artifact:

- resume
- portfolio
- proposal
- slide deck
- report
- poster
- showcase page

### Pain-point query

Names the problem:

- mistake
- rejected
- review comments
- low score
- cannot finish
- how to write

### Reference query

Names a style or precedent:

- template
- example
- before after
- case study
- award-winning

## Screening notes

For each query group, keep only a few high-signal notes. Avoid collecting many near-duplicates.

Track:

- query
- note URL
- reason kept
- reason not yet trusted

## Translation and localization

Use the language that the platform community actually uses. Literal translation is often weaker than community terms.

For example, a formal phrase may perform poorly while a colloquial phrase finds better examples. Record both when comparing.
