# Prompt Rules

Use this reference when writing prompts for design competition drafts.

## Start with the branch

Name the branch first:

- product or poster board
- script or digital media
- environment or spatial design
- video or virtual production

Prompts drift when the branch is unclear.

## Define the object before generating

For a product, board, poster, scene, or video frame, specify:

- what it is
- structure and component count
- size or aspect ratio
- material or medium
- required visual elements
- required text
- forbidden text
- what it must not become

Words like "elegant", "premium", and "cultural" are not enough.

## Product image prompts

Use product images to fix the object's appearance, not to make the final board.

Default constraints:

- one object or product set per prompt
- plain background unless a scene is needed
- main view plus a small number of secondary views
- no ecommerce callouts
- no dimension arrows unless required
- no long text blocks

## Board or poster prompts

State clearly whether this is:

- final competition board
- process board
- product detail board
- poster
- social preview image

Then define:

- page size or aspect ratio
- grid or module layout
- hierarchy
- fixed product facts
- text whitelist
- logo or organizer elements only if verified

Do not ask for "a beautiful poster" when the deliverable is a structured competition board.

## Text whitelist

List every allowed text string:

- exact string
- where it appears
- how many times it may appear
- rough hierarchy or size

Everything outside the whitelist should be banned. This prevents fake labels, placeholder words, and random English.

## Reference roles

Assign each reference one role:

- layout
- type style
- material
- cultural source
- product appearance
- color
- anti-example

When references conflict, preserve factual deliverable definitions first, then layout.

## Common negative constraints

Add task-specific negatives to this list:

```text
fake logo, random English, placeholder text, watermark, ecommerce label,
dimension arrows, unrelated mascot, extra product version, inconsistent color,
wrong category, unreadable small text, copied reference content, collage look,
presentation template look
```

## Practicality

For product or board entries, judges usually need to understand:

- what the core object is
- how the visual system extends
- where it is used
- why it belongs in the track

If a prompt cannot answer those points, narrow the concept before generating.
