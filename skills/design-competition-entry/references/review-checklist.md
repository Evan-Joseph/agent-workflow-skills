# Review Checklist

Use this before regenerating, exporting, or submitting.

## Rules

- [ ] Competition and track name verified.
- [ ] Verification date recorded.
- [ ] Official source recorded.
- [ ] Relative dates converted to exact dates.
- [ ] Deliverables verified.
- [ ] Size, duration, resolution, and file format verified.
- [ ] Open questions marked instead of guessed.

## Branch

- [ ] The selected branch is stated.
- [ ] The current output belongs to that branch.
- [ ] Other branches are not being mixed in by accident.

## Concept

- [ ] One main direction is selected.
- [ ] Source idea is clear.
- [ ] Intended use or experience is clear.
- [ ] Ruled-out directions are recorded.

## Deliverable definition

- [ ] Category is recognizable at first glance.
- [ ] Structure and component count are fixed.
- [ ] Materials or medium are clear.
- [ ] Required text is listed.
- [ ] Forbidden text is listed.
- [ ] The draft has not turned into another category.

## References

- [ ] Each reference has a role.
- [ ] Layout references do not override product facts.
- [ ] Cultural or source references do not add unverified claims.
- [ ] No copied logos, marks, or protected assets appear unless allowed.

## Text

- [ ] All visible text is in the whitelist.
- [ ] No placeholder words remain.
- [ ] No random English or fake labels remain.
- [ ] Text is readable at final size.

## Visual quality

- [ ] Hierarchy is clear.
- [ ] The page does not look like a generic slide template.
- [ ] The draft does not look like an ecommerce listing unless that is intended.
- [ ] Main object and supporting details are balanced.
- [ ] The work looks complete enough for the competition format.

## Submission

- [ ] Final files use the required format.
- [ ] File names are clear.
- [ ] Source and final assets are separated.
- [ ] Platform fields match local files.
- [ ] Optional upload slots do not contain stale files.
- [ ] Submission status has been checked after upload if platform access is available.
