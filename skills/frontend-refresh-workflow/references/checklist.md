# Frontend Refresh Checklist

## Takeover

- [ ] Git status checked.
- [ ] Frontend and backend directories identified.
- [ ] Run commands found and tested.
- [ ] Required environment variables identified.
- [ ] Ports checked.
- [ ] App opened in a browser.

## Current UI map

- [ ] Routes and entry points confirmed.
- [ ] Main pages documented.
- [ ] Important content blocks documented.
- [ ] Buttons, forms, filters, pagination, dialogs, uploads, maps, media players, and navigation documented.
- [ ] Data sources and API calls documented.
- [ ] Auth requirements documented.
- [ ] Loading, empty, error, disabled, selected, expanded, and unauthenticated states documented.
- [ ] Real behavior separated from mocks and placeholders.

## Prototype brief

- [ ] Page scope is clear.
- [ ] Required content blocks are clear.
- [ ] Required states are clear.
- [ ] Visual constraints are clear.
- [ ] The brief is tool-neutral unless the user asked otherwise.

## Prototype integration

- [ ] Original prototype or archive preserved.
- [ ] Framework and package manager identified.
- [ ] Routes mapped to product pages.
- [ ] Dependencies installed with the project's package manager.
- [ ] New frontend placed in a clear directory.
- [ ] Old frontend archived only if needed.

## Real behavior

- [ ] Auth flow works.
- [ ] Forms submit to real handlers.
- [ ] Lists use real data.
- [ ] Search, filters, and pagination work.
- [ ] Upload, map, player, or third-party features work if kept.
- [ ] Mock data removed or clearly documented as fixtures.
- [ ] Dead controls removed.

## Verification

- [ ] Start commands tested.
- [ ] Main routes opened.
- [ ] Success path tested.
- [ ] Error path tested.
- [ ] Empty state tested.
- [ ] Browser console checked.
- [ ] README updated.
- [ ] Environment example updated.

## Final check

- [ ] No fake buttons.
- [ ] No unexplained mock data.
- [ ] No dead routes.
- [ ] Every kept control has a reason to exist.
- [ ] A new developer can run the project from the README.
