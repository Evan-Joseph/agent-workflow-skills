# Frontend Prototype Brief

Build a new frontend prototype for an existing project.

This brief describes the product, pages, states, interactions, and visual constraints. It does not assume a specific AI frontend tool or handoff format.

## Goal

Create a runnable frontend prototype that can later be connected to the existing backend. Do not make a static mockup.

## General requirements

- Keep a consistent visual language.
- Include all required pages.
- Make page-to-page navigation work.
- Give every major control a purpose.
- Cover normal, loading, empty, error, disabled, selected, and expanded states where relevant.
- Keep desktop and mobile layouts usable.

## Do not do

- Do not make screenshot-only pages.
- Do not omit dialogs, empty states, error states, or loading states.
- Do not make visible controls that cannot be connected to real behavior later.
- Do not remove required pages.
- Do not add unrelated product features.
